Windows 版本打包脚本（file_change）

脚本简介

该脚本用于交叉编译 Go + Fyne 开发的 file_change 应用，目标平台为 Windows 64位（Windows 10/11）。核心特色是每次打包随机嵌入本地图标目录中的 ICO/PNG 图标，同时具备容错机制（Fyne 打包失败时自动切换备用编译方案），确保产物可用。

核心配置

配置项 说明 取值
PROJECT_DIR 项目根目录 $HOME/GO_Pro/file_change
APP_NAME 输出 EXE 文件名 file_change.exe
OUTPUT_DIR 产物存放目录 $PROJECT_DIR/dist/windows
ICON_DIR 本地图标文件夹（需手动确认路径） $HOME/icon/win_icons
GO_BIN_PATH Go 二进制文件路径 /usr/local/go/bin

交叉编译固定参数

参数 说明 取值
WINDOWS_GOOS 目标系统 windows
WINDOWS_GOARCH 目标架构（64位） amd64
WINDOWS_CC 交叉编译器 x86_64-w64-mingw32-gcc

动态路径（自动获取）

  • GOPATH：从环境变量获取，取第一个路径
  • FYNE_CMD：Fyne 打包工具路径（$GOPATH/bin/fyne）
  • EXECUTABLE_PATH：编译输出路径（$PROJECT_DIR/$APP_NAME）

打包流程

1. 环境信息输出

脚本启动时打印关键信息：

  • Go 版本
  • 目标平台（Windows 64位）
  • 本地图标目录路径
  • 最终产物路径
  • 核心功能（随机嵌入图标）

2. 依赖与图标检查（核心步骤）

自动校验打包必需条件，避免后续失败：

  1. 检查 Fyne 工具：若未安装，提示执行 go install fyne.io/fyne/v2/cmd/fyne@latest
  2. 检查图标目录：确认 ICON_DIR 路径存在，否则报错提示
  3. 收集可用图标：优先读取 ICO 格式图标，再补充 PNG 格式（存入数组）
  4. 随机选择图标：从图标数组中随机抽取一个作为本次打包的嵌入图标
    • 若图标目录无可用图标（无 ICO/PNG），脚本直接退出并提示

备用方案：手动编译 + 图标嵌入（Fyne 失败时触发）

    • 安装 Go 环境（版本 ≥ 1.16）
    • 安装 Fyne 工具：go install fyne.io/fyne/v2/cmd/fyne@latest

    • 双击即可运行（图标已内嵌，无需额外文件）

注意事项

  1. 系统兼容性：

    • 脚本使用 dnf 包管理器，适用于 Fedora、RHEL 等系统
    • 若使用 Ubuntu/Debian，需将 dnf install 替换为 apt-get install -y，依赖包名称调整为 mingw-w64 gcc-mingw-w64 binutils-mingw-w64

  2. 图标要求：

    • 支持 ICO（推荐）和 PNG 格式，建议尺寸 256x256（确保 Windows 下显示清晰）
    • 图标目录中可存放多个图标，脚本每次自动随机选择

  3. 关键依赖：

    • 必须安装 Fyne 工具：go install fyne.io/fyne/v2/cmd/fyne@latest
    • 交叉编译依赖 mingw64-gcc 和 mingw64-binutils，脚本会自动安装

  4. CGO 禁用说明：

    • 脚本默认禁用 CGO（CGO_ENABLED=0），避免 Windows 下链接器兼容性问题，若应用依赖 CGO 功能，需手动改为 CGO_ENABLED=1 并调整编译参数

  5. 产物特性：

    • 生成的 EXE 为独立文件，无需额外依赖（Go 静态编译）
    • 图标已内嵌，Windows 下显示正常（无默认图标）
